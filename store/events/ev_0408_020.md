在服务器上跑 cc 隔离是完全可行的，而且思路很合理。几个方案：

**方案一：Docker 容器（推荐）**

最干净的隔离方式。每次用完销毁容器，机器指纹随之消失。

```dockerfile
FROM node:20-slim
RUN npm install -g @anthropic-ai/claude-code
WORKDIR /workspace
```

```bash
docker run -it --rm \
  -v $(pwd):/workspace \
  -e ANTHROPIC_API_KEY=xxx \
  claude-code-image
```

每次 `--rm` 确保容器无状态，不留痕迹。

**方案二：你已有的 SF VPS 直接用**

你的 DigitalOcean VPS 本来就是干净环境，直接在上面：
- 用 `code-server` 你已经配好了
- cc 装在 VPS 上跑，机器标识是服务器的，和本地电脑完全隔离
- Git config 单独配一个项目级别的（`git config --local`），不暴露全局 email

**方案三：专用系统用户**

```bash
sudo useradd -m cc-user
sudo su - cc-user
npm install -g @anthropic-ai/claude-code
```

新用户有独立的 home 目录、独立的 git config，和你主账户的邮箱/环境完全分开。
