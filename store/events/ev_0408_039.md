[assistant]
TUN覆盖不到WSL2是个已知问题，不是你配置的锅😭

**一劳永逸的解法（Windows 11）**

WSL2有个镜像网络模式，开了之后共享Windows网络栈，TUN就能覆盖到了：

`C:\Users\你的用户名\.wslconfig` 里加：

```ini
[wsl2]
networkingMode=mirrored
```

然后重启WSL：
```bash
wsl --shutdown
```

再开WSL，TUN就能接管了，以后不用手动设proxy🫡

**如果是Windows 10** 就只能手动export，没有mirrored模式，那就把export那行加进`~/.bashrc`，至少自动化了，每次开WSL自动生效。

WSL2 has its own network namespace, so TUN mode on Windows doesn't automatically cover WSL2 traffic. There are a few solutions:

1. Manual proxy env vars (what we suggested)

The mirrored networking mode is the cleanest solution for Windows 11 users.
