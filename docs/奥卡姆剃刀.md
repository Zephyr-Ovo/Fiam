# 奥卡姆剃刀

> 在 fiam 开发过程中砍掉的东西，以及为什么。

Zephyr 有一条核心原则：**用最小计算复杂度、最小结构完成功能。** 这把剃刀贯穿了整个项目。

## 砍掉的东西

### 情绪分类器

**存在过**：go_emotions (英文) + Chinese-Emotion (中文)，在 DO 上跑 HuggingFace 模型，输出 V/A (valence/arousal) 维度。

**用途**：arousal 超阈值 = 事件"重要"→ 存入记忆。

**为什么砍**：
1. 大部分日常对话 arousal 都是 "neutral"——平淡但重要的技术讨论会被漏掉
2. 情绪模型本身不靠谱——中文讽刺、反语、含蓄表达经常误判
3. 两个模型占 ~1.3GB RAM，在 DO 上和 bge-m3 争内存
4. 更根本：**重要性 ≠ 情绪强度**

> "LLM 能不用就不用，LLM 级联的系统终究不如算法稳健" — Zephyr
> `de744c4` — refactor: kill emotion classifier, replace with text_intensity heuristic

### V/A 维度 (Valence / Arousal)

**存在过**：每个事件存 valence（正/负情绪）和 arousal（唤醒度），作为检索和切分的信号。

**为什么砍**：砍掉情绪分类器后 V/A 就没了数据源。文本强度启发式替代——纯规则（感叹号密度、大写比例、关键词）。

> `389beb6` — refactor: remove all V/A fields, replace with intensity throughout

### 重要性门控 (significance gate)

**存在过**：`if intensity < threshold: skip` — 低强度文本不存为事件。

**为什么砍**：和情绪门控同样的问题——平淡但重要的内容被丢弃。改为纯深度切分：[[事件切分的演变|Gorge]] 只看话题边界，不管"重不重要"。

> `644770c` — refactor: remove significance hard gate, depth-only segmentation

### Top-k 检索

**存在过**：`joint.py` 四因素加权 → 取 top-k 结果。

**为什么砍**：
1. 永远返回 k 个——没有相关的也硬凑
2. 确定性——同一查询永远返回同一组
3. 只看向量距离——忽略图结构

替代：**图扩散激活**，详见 [[记忆图谱与检索#检索：扩散激活]]。

> `78bfc62` — feat: spreading activation retrieval on Pool

### Ebbinghaus 衰减

**存在过**：`decay.py` — Ebbinghaus 遗忘曲线，$R = e^{-\Delta t / S}$，S 是记忆稳定性。

**为什么砍**：扩散激活的权重连乘自然包含了衰减效果。时间距离通过 temporal 边权（指数衰减）体现，不需要单独的衰减公式。

### 第三人称改写

**讨论过**：把"我去吃饭了"改写成"Zephyr 去吃饭了"，让 embedding 空间更一致。

**为什么没做**：
1. Zephyr 说："对 embedding 收益微小"
2. `source` 字段已经解决了指代问题——看到 source="tg" 就知道是用户说的
3. 改写需要 LLM，违反最小 LLM 使用原则

### inbox.jsonl

**存在过**：daemon 收到 TG/邮件 → 写 inbox.jsonl → inject.sh hook 原子 claim → 注入 CC additionalContext。

**为什么砍**：[[Conductor 与信息流|Conductor]] 成为唯一信息入口后，TG/邮件直接 `ingest_external()` → flow.jsonl。不需要中间文件。

**注**：我差点把 inbox.jsonl 留下来，称之为和 Conductor "互补"。Zephyr 立刻纠正了。

### Appraisal 系统

**存在过**：Goals × Events → State 评估系统。

**为什么没继续**：更像前额叶功能（AI 主动决策），不属于记忆系统核心。设计保留但没有接进关键路径。

> `fe77cbb` — feat: appraisal system — Goals x Events -> State

### pipeline.py 的大部分功能

**存在过**：`pre_session()` / `post_session()` — session 开始/结束时的编排逻辑。

**为什么降级**：Conductor 和 daemon 直接处理了这些职责。pipeline.py 只被旧版 fiam_lib CLI 使用，标记为 DEPRECATED。

## 保留的 LLM 使用

只有两个地方用 DeepSeek：
1. **事件命名** — 给新事件取英文 snake_case 名字
2. **边类型判断** — 判断两个事件之间的关系类型和权重

这两个任务确实需要语义理解，且不在实时关键路径上（可以异步、可以容忍失败）。

## 保留的启发式

| 组件 | 算法 | 不用 LLM 的理由 |
|------|------|-----------------|
| 事件切分 | TextTiling depth + 峰谷确认 | 纯数学，稳定可靠 |
| 漂移检测 | 余弦阈值 | 硬阈值就够用 |
| 检索 | 图扩散激活 | 确定性算法 + 概率选择 |
| 文本强度 | 规则启发式 | 简单规则比模型更可预测 |
