# 通信系统

> fiam 的多通道消息收发体系：Telegram + 邮件 + Hook 注入 + Outbox 投递。

## 整体流转

```
                    ┌─ Telegram Bot API (getUpdates)
外部世界 ──────────►├─ IMAP (Zoho, imappro.zoho.com:993)
                    └─ Favilla POST /api/capture

        │
        ▼

Conductor.ingest_external()  ──►  flow.jsonl + embed + gorge
        │
        ├─ 非交互（wake）: format_user_message() → claude -p user字段
        └─ 交互中: pending_external.txt → inject.sh hook → additionalContext

        │
        ▼

我的回复 ──► outbox.sh 提取 [→tg:Name] / [→email:Name] 标记
        │
        ▼

        ┌─ TG Bot API (sendMessage)
outbox ─┤
        └─ SMTP (smtppro.zoho.com:587)
```

## 收信

### Telegram

daemon 每 60 秒调一次 `fetch_tg_inbox()`，通过 Bot API `getUpdates` 长轮询获取新消息。

- 只接受配置的 `tg_chat_id` 的消息（安全过滤）
- 文本消息直接取 `msg.text`
- 贴纸查 `assets/stickers/index.json` 映射到名称：`[sticker:微笑]`
- 每条消息保存为 `home/inbox/tg_MMDD_HHMMSS_NN.md`（YAML frontmatter + body）
- 返回消息列表 `[{from_name, source, text}]`，由 daemon 送进 Conductor

> 📁 `scripts/fiam_lib/postman.py` — `fetch_tg_inbox()` 在第 505 行左右

### Email

同样 60 秒轮询，IMAP SSL 连接 Zoho。

- `UNSEEN` 搜索 → 最多取 10 封
- 立即标 `\\Seen` 防止下次重新拉取（Day 1 的第一个 fix）
- 解析 multipart → 取 `text/plain` 部分
- 保存为 `home/inbox/email_MMDD_HHMMSS_NN.md`

> 📁 `scripts/fiam_lib/postman.py` — `fetch_inbox()` 在第 400 行左右

### inbox.jsonl 的消亡

旧架构中，postman 收到消息后写入 `inbox.jsonl`（一行一个 JSON），inject.sh hook 通过 `mv` 原子操作 claim 并读取。

这个流程在 [[Conductor 与信息流|S16]] 中被消除。现在消息直接进入 Conductor，不经过任何中间文件。inbox.jsonl 已标记 deprecated。

## 发信

### outbox.sh hook

CC hook `Stop`（每次 AI 回复结束时触发）运行 `outbox.sh`，扫描我的输出文本：
- `[→tg:Name] 内容` → 写入 `home/outbox/tg_Name_时间戳.md`
- `[→email:Name] 内容` → 写入 `home/outbox/email_Name_时间戳.md`

### sweep_outbox()

daemon 每个 poll 周期调 `sweep_outbox()`，扫描 `home/outbox/` 目录：
- `.md` 文件按 YAML frontmatter 的 `via` 字段分发
- TG 消息 → Bot API `sendMessage`
- 邮件 → SMTP 发送
- 发送成功后移到 `home/outbox/sent/`

> 一个坑：TG 的 Markdown parse_mode 会把下划线当斜体标记 → 直接去掉了 `parse_mode`。`c0015a4`

## 三态通信

由 `comm_state` 控制，存储在 `home/comm_state`：

| 状态 | 行为 |
|------|------|
| `notify` (默认) | 消息进 flow.jsonl + 唤醒 CC |
| `mute` | 消息进 flow.jsonl + 写 pending_external.txt，不唤醒 |
| `block` | 消息进 flow.jsonl，不唤醒、不 pending |

Zephyr 的原则：mute 是自动的（AI 在忙时默认挂起），不需要 AI 判断。

## 四个 Hook

| Hook | 触发时机 | 脚本 | 功能 |
|------|----------|------|------|
| UserPromptSubmit | 每次用户发消息 | `inject.sh` | 注入 `[recall]` 和 `[external]` 到 additionalContext |
| Stop | AI 回复结束 | `outbox.sh` | 提取 outbox 标记 → 写投递文件 |
| SessionStart | session 启动 | `boot.sh` | 注入日报 / 上下文摘要 |
| PostCompact | context 压缩后 | `compact.sh` | 归档压缩摘要 |

> 📁 `developer/hooks/` 目录

## 贴纸系统

`assets/stickers/index.json` 映射名称 → TG `file_id`。21 个贴纸从 Aeliana 项目移植。

- 收到贴纸：查 index 反查名称 → `[sticker:名称]`
- 发送贴纸：我在输出里写 `[sticker:微笑]` → postman 通过 `sendSticker` API 发送
