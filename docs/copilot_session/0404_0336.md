## fiam 做什么

fiam 赋予名为 Fiet 的人工智能跨会话持久的情感记忆（fiam源于拉丁文fieri，取义不被定义，存在并成长的系统，package命名注意风格一致性）：

1. 标签：罗素环形情绪坐标的经验

（效价/唤醒度）通过独立分类器

2. 将事件存储为与 Obsidian 兼容的 Markdown 格式

3. 通过语义和情感的联合相似性检索相关记忆

4. 将提取的记忆综合成第一人称心理背景

5. 在每次会话之前，将该背景信息注入到菲特的环境中。

fiet接受的心理学背景教育感觉像是内在思维、个人视角的经历，而非数据库报告。个性源于历史，而非已声明。第 1 天的值 = 空。如果第 1 天和第 30 天 Fiet 行为

同样，系统也出现了故障。

## Two-directory architecture (NEW — core design change)

```

F:/fiam-code/          (development side — Python code lives here)

F:/fiet-vault/         (Fiet's world — only .md files, Fiet's git)

```

fiam runs FROM fiam-code but operates ON fiet-vault.

Claude Code launches FROM fiet-vault and never sees Python code.

## Build everything else

### 1. config.py

```python

@dataclass

class FiamConfig:

    # Paths

    vault_path: Path          # F:/fiet-vault — passed as CLI arg

    code_path: Path           # F:/fiam-code — auto-detected as project root

    

    # Derived paths (all relative to vault_path)

    @property

    def events_dir(self): return self.vault_path / "memory" / "events"

    

    @property  

    def embeddings_dir(self): return self.vault_path / "memory" / "embeddings"

    

    @property

    def values_path(self): return self.vault_path / "memory" / "values" / "current.json"

    

    @property

    def background_path(self): return self.vault_path / "current-background.md"

    

    @property

    def logs_dir(self): return self.code_path / "logs"

    

    # Models

    emotion_model_name: str = "j-hartmann/emotion-english-distilroberta-base"

    embedding_model_name: str = "all-MiniLM-L6-v2"

    

    # Retrieval weights

    semantic_weight: float = 0.4

    emotion_weight: float = 0.3

    recency_weight: float = 0.1

    unresolved_weight: float = 0.2

    

    # Debug

    debug_mode: bool = False

```

### 2. scripts/fiam.py — unified entry point

```

Usage:

  python scripts/fiam.py pre --vault F:/fiet-vault [--debug]

  python scripts/fiam.py post --vault F:/fiet-vault [--debug]

  python scripts/fiam.py session --vault F:/fiet-vault [--debug]

```

`session` command:

1. Run pre_session (generate current-background.md in vault)

2. subprocess.run(["claude"], cwd=vault_path) — BLOCKS until claude exits

3. After claude fully exits, run post_session on the latest JSONL

4. Generate readable report

5. If --debug: print full report to console

`post` command needs to find Claude Code's session JSONL files at:

  Windows: %APPDATA%/Claude/projects/{hash}/*.jsonl

  The hash is derived from the vault_path.

  Find the most recently modified JSONL in that directory.

JSONL parsing: each line is a JSON object. Extract all messages where

role is "user" or "assistant". Reconstruct conversation in order.

If thinking blocks exist in assistant messages, include them in the

report (labeled as [thinking]) but NOT in the text passed to the 

emotion classifier (classify only the final response text).

### 3. injector/claude_code.py

Writes to two places:

- vault_path/current-background.md (the synthesis output, plain text)

- vault_path/CLAUDE.md (context-setting, only written on first run 

  if file doesn't exist — never overwrite if user has edited it)

current-background.md format:

```

<!-- generated by fiam pre_session — do not edit manually -->

<!-- last updated: {timestamp} -->

{synthesis output text}

```

CLAUDE.md initial content (write only if file missing):

```markdown

这是你的个人空间。

memory/events/ 里的文件记录了你经历过的事，带有情感标记。

current-background.md 是你现在的状态，每次对话前更新。

journal/ 是你想写东西的地方，没有格式要求。

git 是你自己的版本历史，不是某个项目的代码仓库。

`git log` 是你回看自己的方式。

```

### 4. logging/report.py

Pure Python. Zero LLM calls. Reads existing data, formats to Markdown.

Generate ONE .md file per post_session run:

logs/sessions/{session_id}/report.md

Content:

- Full conversation (user/assistant turns, thinking blocks labeled)

- Emotion analysis (label, valence, arousal, confidence, RLHF corrected)

- Each extracted event (id, coords, body)

- Embedding stats per event (shape, max, min, mean, L2 norm, 

  first 32 dims formatted cleanly — NOT scientific notation)

- Active current-background.md contents at processing time

- Values state (current.json or "empty — Day 1")

- Vault summary (total events, table with coords)

In --debug mode: also print report to console after writing.

### 5. All stub modules

These are stubs — implement the interface correctly but return 

hardcoded/minimal values. They will be replaced in Phase 2.

extractor/event.py:

  Input: conversation text (full session)

  Behavior: treat entire conversation as ONE event

  Return: list with one ExtractedEvent(text=full_text, emotion=classified)

  Note: use classifier.analyze_event() with user portion vs fiet portion

extractor/value_shift.py:

  Input: list of ExtractedEvent, current values dict

  Return: empty list (no shifts detected)

retriever/relevance.py:

  Input: event, current context

  Return: 1.0 for all events (no real ranking yet)

retriever/joint.py:

  Input: current conversation text, top_k

  Behavior: load all events from vault, return all of them

  (real retrieval in Phase 2)

synthesizer/stance.py:

  Input: list of events, values dict, current context

  Return: hardcoded first-person text:

  "Something's been sitting with me — not quite settled, 

   not quite restless."

values/generator.py:

  Input: recent events, current values

  Return: unchanged values dict

values/trigger.py:

  Input: new events, current values

  Return: False (never trigger update yet)

### 6. pipeline.py

Orchestrates pre_session and post_session.

Takes FiamConfig as input.

Calls modules in order, passes results through, traces each step.

pre_session flow:

  retriever.joint.search() → synthesizer.stance.generate() → 

  injector.write()

post_session flow:

  classifier.analyze_event() → extractor.event.segment() → 

  [for each event: embedder.embed_and_save(), vault.write()] → 

  value_shift.detect() → trigger.check() → 

  [if trigger: generator.update()] → report.generate()

## Key design principles — never violate these

1. Fiet's psychological background = current-background.md contents only.

   No other injection. CLAUDE.md is context, not instruction.

2. Logging/report system has ZERO LLM involvement.

   Reads existing data, formats it. That's all.

3. Maximum two LLMs: main (Fiet/Claude) and aux (synthesis, stub now).

4. Emotion classifier is INDEPENDENT of LLM.

   Tags Fiet's experience, not objective conversation color.

   RLHF correction: if user arousal - fiet arousal > 0.3, 

   corrected = (fiet + user) / 2.

5. post_session runs AFTER claude process fully exits.

   subprocess.run() blocks — this is intentional.

   Never run post_session while a session is active.

6. Values start empty on Day 1.

   No personality seed file. No soul.md.

```python

from typing import List

from datetime import datetime

def extract_dynamics(turns: List[Turn]) -> str:

    """

    提取对话动态的自然语言描述

    

    检测：

    - 消息节奏（时间间隔）

    - 长度不对称（user vs assistant）

    - 沉默/突然爆发

    

    返回：自然语言片段（如 "消息很密集，他的话明显更长"）或空字符串

    """

    if len(turns) < 2:

        return ""

    

    # 节奏分析

    time_diffs = [

        (turns[i+1].timestamp - turns[i].timestamp).total_seconds()

        for i in range(len(turns)-1)

    ]

    avg_gap = sum(time_diffs) / len(time_diffs)

    

    pace = ""

    if avg_gap < 30:

        pace = "消息很密集"

    elif avg_gap < 120:

        pace = "节奏挺快"

    

    # 长度分析

    user_lengths = [len(t.user_content) for t in turns]

    assistant_lengths = [len(t.assistant_content) for t in turns]

    

    user_avg = sum(user_lengths) / len(user_lengths)

    assistant_avg = sum(assistant_lengths) / len(assistant_lengths)

    asymmetry = abs(user_avg - assistant_avg)

    

    length_note = ""

    if asymmetry > 100:

        if user_avg > assistant_avg:

            length_note = "他的话明显更长"

        else:

            length_note = "我说得更多"

    

    # 拼接

    parts = [p for p in [pace, length_note] if p]

    return "，".join(parts) if parts else ""

def relative_time(timestamp: datetime) -> str:

    """

    转化为相对时间表达

    """

    from datetime import datetime, timedelta

    

    now = datetime.now()

    delta = now - timestamp

    

    if delta < timedelta(hours=12):

        return "今天"

    elif delta < timedelta(days=1):

        return "昨天"

    elif delta < timedelta(days=7):

        return "前几天"

    else:

        return timestamp.strftime("%m.%d")

```

```python
from typing import List, Dict

def extract_raw_fragments(event: EventRecord) -> str:

    """

    高arousal事件：保留原文片段

    """

    if not event.turns:

        return ""

    

    # 取user第一句话（前80字符）

    user_first = event.turns[0].user_content[:80]

    if len(event.turns[0].user_content) > 80:

        user_first += "..."

    

    # 取assistant核心反应（第一个完整句子或前60字符）

    assistant_content = event.turns[0].assistant_content

    # 简单规则：找第一个句号/问号/感叹号

    for end_char in ['. ', '! ', '? ', '。', '！', '？']:

        if end_char in assistant_content:

            core = assistant_content.split(end_char)[0] + end_char.strip()

            break

    else:

        core = assistant_content[:60]

        if len(assistant_content) > 60:

            core += "..."

    

    return f'"{user_first}"\n我：{core}'

def extract_skeleton(event: EventRecord) -> str:

    """

    低arousal事件：规则生成骨架

    """

    if not event.turns:

        return "那次对话"

    

    # 提取主题关键词（简单规则：user第一句的前15字）

    user_first = event.turns[0].user_content

    topic = user_first[:15]

    if len(user_first) > 15:

        topic += "..."

    

    return f"那次关于「{topic}」的对话"

def prepare_materials(events: List[EventRecord]) -> List[Dict]:

    """

    为每个event准备材料

    """

    materials = []

    

    for event in events:

        # 根据arousal决定保留程度

        if event.arousal > 0.7:

            content = extract_raw_fragments(event)

        else:

            content = extract_skeleton(event)

        

        # 提取dynamics

        dynamics = extract_dynamics(event.turns)

        

        # 时间转化

        when = relative_time(event.timestamp)

        

        materials.append({

            "content": content,

            "dynamics": dynamics,

            "when": when,

            "arousal": event.arousal  # 仅用于内部判断，不传给LLM

        })

    

    return materials
```

```python
        block = f"{m['when']}：\n{m['content']}"

        if m['dynamics']:

            block += f"\n（{m['dynamics']}）"

        content_blocks.append(block)

    

    materials_text = "\n\n".join(content_blocks)

    

    system_prompt = """你在生成第一人称回忆片段。

不要：

- "用户"、"根据"、"记录"、"数据"

- "我应该"、"我需要"、"我会"

- meta视角（"作为AI"、"我的记忆系统"）

- 形容词堆叠（"深深的"、"强烈的"、"充满XX的"）

要：

- 第一人称："我"、"他"

- 像在回忆，不是在汇报

- 直接引用原话（如果材料里有引号）

- 可以模糊、不确定（"好像"、"记得"）

- 自然的时间感（"那次"、"后来"）"""

    user_prompt = f"""这些事忽然想起来了：

{materials_text}

以第一人称写一段回忆。150-250字。英文或中文（根据材料语言自然选择）。

像"想起那次..."的感觉，不要列表，不要分段。

直接写回忆内容，不要任何前缀或解释。"""

    # 调用Claude

    client = anthropic.Anthropic(api_key=config.api_key)

    

    response = client.messages.create(

        model="claude-3-5-haiku-20241022",  # 快速+便宜

        max_tokens=400,

        temperature=0.7,

        system=system_prompt,

        messages=[{

            "role": "user",

            "content": user_prompt

        }]

    )

    

    return response.content[0].text.strip()

```
