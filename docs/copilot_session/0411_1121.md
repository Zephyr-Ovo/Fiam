## 论文参考（2026-04-09 整理）

### LLM 情感数理研究

- **"Emotions Where Art Thou"** (2510.22042): LLM 内部低维情感流形，方向编码，跨 5 语言对齐。给 WDI 方法论背书。

- **"Whether, Not Which"** (2603.22295): affect reception (AUROC 1.0) vs emotion categorization (部分依赖关键词) 两个可分离机制。有代码。

- **"E-STEER"** (2604.00005): emotion steering 框架，情感-行为非单调关系。和 Anthropic 情感论文交叉验证。

- **"Emotion in SLMs"** (2604.04064): 124M-3B 模型情感表征在 ~50% depth 中间层，U 形曲线 architecture-invariant。Qwen 跨语言情感纠缠。

- **"LLM Assertiveness"** (2508.17182): 自信度分解为 emotional + logical 正交子空间。情感维度和逻辑维度解耦。

- **Anthropic Apr 2026**: 情感向量因果性驱动行为（blackmail/reward hacking），pretraining 继承 + post-training 塑形。

### 记忆架构

- **"DAM"** (2512.21567): 记忆管理 = 不确定性下序贯决策，value function + uncertainty estimator。fiam 权重从 heuristic → learned policy 的理论路径。值得深读。

- **"SEEM"** (2601.06411): 图关系层 + 动态情景层，Episodic Event Frames + provenance pointer 溯源。fiam 可加 source_session_id。

- **"FadeMem"** (2601.18642): 自适应指数衰减（semantic + access + temporal 多因子调制），45% 存储压缩。

- **"HyMem"** (2602.13933): 双粒度存储 + 动态两级检索，比 full-context 减 92.6% 计算。

- **"SYNAPSE"** (2601.02744): spreading activation 图检索 + lateral inhibition + temporal decay，LoCoMo SOTA。

- **"Mem0"** (2504.19413): production baseline，graph memory +2%, 比 OpenAI +26%。benchmark 对比对象。

- **"MemMachine"** (2604.04853): 保留原始对话 + contextualized retrieval。核心发现：检索阶段优化 >> 摄入阶段优化。

- **"MemFactory"** (2603.29493): 原子化可插拔记忆组件 + GRPO 强化学习微调。有代码。

- **"AI Hippocampus"** (2601.09113, TMLR): 最全 survey，implicit/explicit/agentic 三分法。related work 参考。

- **"StructMemEval"** (2602.11243): 测试记忆组织能力而非事实召回。

## 与 Claude 讨论暂存（2026-04-09）

### Action Preference Learning — 不做

- action 粒度太粗（"写东西"本身不携带信息），信号在 action × context × user state 组合

- 学 action preference 学到的是幻觉，真正信号在"用户处于什么状态时，什么内容让他兴奋"

- 结论：别在 Phase 1 加 action 字段，等图结构上来自然浮现

### 因果性方向

- **Pearl 因果阶梯**: 观察→干预→反事实。所有 RAG 系统停在第一级

- **Granger 因果性**: 知道 A 的历史能更好预测 B → A Granger-causes B。fiam event 上可做但量少噪声大

- **Schema Theory**: 人存抽象"脚本"而非原始序列（"每次和 X 见面前我都焦虑"）→ 图边权重积累

- **最大价值**: 解释情绪来源而非预测行为。"你上次这么说是答辩前一天" = 质变

### LangGraph

- LangChain 的图状 agent 编排框架，节点=动作，边=流转条件，原生状态持久化+循环

- fiam 作为 daemon 用不上，适合 multi-step workflow。了解就好。

### 跨 Event 模式 → Phase 2 HippoRAG

- 单条 event 是原子，图上路径才是模式

- action preference 作为孤立特征没用，但作为图边权重有意义

DERO上下文里被提及，反复出现且分布在不同的上下文

里，这个词才开始获得独立于任何单一语境的稳定

权重。

果我变了，它也会跟着变。
