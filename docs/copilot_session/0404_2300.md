## fiam 项目交接

### 是什么

fiam 是一个 **bio-inspired 记忆系统**，为 AI 人格 "Fiet"（运行在 Claude 上）提供跨会话的情感记忆、价值观追踪和背景注入。它不是聊天应用本身，而是围绕 Claude Code 会话的**前处理/后处理管道**。

### 双目录架构

- **fiam-code** — Python 代码，用 `uv` 管理（不是 conda/pip）

- **fiet-vault** — Obsidian vault，存放 Fiet 的记忆产物（`.md` 事件文件、embedding `.npy`、`current-background.md`、`journal/`）

代码**永远不写进 vault**，vault 里**只有 .md 和 .npy**。

### 运行方式

```

uv run python scripts/fiam.py pre  --vault F:/fiet-vault [--debug]   # 会话前：检索→合成→注入背景

uv run python scripts/fiam.py post --vault F:/fiet-vault [--debug]   # 会话后：分类→提取→嵌入→存储

uv run python scripts/fiam.py session --vault F:/fiet-vault          # pre → 启动claude → post 全自动

```

fiam.py 里手动 `sys.path.insert(0, src/)`，**没有** `[build-system]` 配置，`fiam` 包不是 pip-installable 的。直接 `import fiam` 在 scripts/ 外面会失败。

### 核心流水线（pipeline.py）

**pre_session**: `joint_retriever.search() → StanceSynthesizer.generate() → injector.write()` → 产出 `current-background.md`

**post_session**: `event_extractor.segment(conversation, classifier)` → 对每个事件 `embedder.embed() + vault.write()` → `value_shift.detect()` → `trigger.check()` → `report.generate()`

### 情感分类器（最近刚改，重点）

**双模型架构**，在 emotion.py：

| 层 | 模型 | 作用 | 为什么需要 |

|---|---|---|---|

| Valence | `cardiffnlp/twitter-xlm-roberta-base-sentiment` | 3-class (pos/neu/neg) → 连续 valence [-1,1] | **多语言**，能抓中文微妙情绪（愧疚→neg 0.91，感动→pos 0.52） |

| Arousal | `j-hartmann/emotion-english-distilroberta-base` | 7-class → Russell circumplex映射 | 英文 arousal 分辨力强（anger→0.7, surprise→0.9） |

| Arousal floor | 文本启发式 `_text_arousal_signal()` | 正则检测 !!!、啊啊啊、哈哈哈、ALL CAPS | 兜底，两个模型对纯符号/重复字都不行 |

**融合规则**：`valence = cardiffnlp`，`arousal = max(j_hartmann, heuristic, sentiment_derived)`

**RLHF 修正**（`analyze_event`）：如果 user arousal 比 assistant arousal 高 > 0.3，把 assistant 的 arousal 往上拉。这是为了补偿 Claude 被 RLHF 训练出的情绪压制倾向。

### 事件提取器（event.py）

**不是所有对话都存**。阈值过滤：

- `arousal > 0.6` → 显著

- `|valence| > 0.5 AND arousal > 0.4` → 显著

- 其余丢弃

对话先分成 user-assistant pairs，每对独立分类，再按相邻性和关键词重叠决定是否合并。

from typing import List

from datetime import datetime

def extract_dynamics(turns: List[Turn]) -> str:

    """

    提取对话动态的自然语言描述

    

    检测：

    - 消息节奏（时间间隔）

    - 长度不对称（user vs assistant）

    - 沉默/突然爆发

    

    返回：自然语言片段（如 "消息很密集，他的话明显更长"）或空字符串

    """

    if len(turns) < 2:

        return ""

    

    # 节奏分析

    time_diffs = [

        (turns[i+1].timestamp - turns[i].timestamp).total_seconds()

        for i in range(len(turns)-1)

    ]

    avg_gap = sum(time_diffs) / len(time_diffs)

    

    pace = ""

    if avg_gap < 30:

        pace = "消息很密集"

    elif avg_gap < 120:

        pace = "节奏挺快"

    

    # 长度分析

    user_lengths = [len(t.user_content) for t in turns]

    assistant_lengths = [len(t.assistant_content) for t in turns]

    

    user_avg = sum(user_lengths) / len(user_lengths)

    assistant_avg = sum(assistant_lengths) / len(assistant_lengths)

    asymmetry = abs(user_avg - assistant_avg)

    

    length_note = ""

    if asymmetry > 100:

        if user_avg > assistant_avg:

            length_note = "他的话明显更长"

        else:

            length_note = "我说得更多"

    

    # 拼接

    parts = [p for p in [pace, length_note] if p]

    return "，".join(parts) if parts else ""

def relative_time(timestamp: datetime) -> str:

    """

    转化为相对时间表达

    """

    from datetime import datetime, timedelta

    

    now = datetime.now()

    delta = now - timestamp

    

    if delta < timedelta(hours=12):

        return "今天"

    elif delta < timedelta(days=1):

        return "昨天"

    elif delta < timedelta(days=7):

        return "前几天"

    else:

        return timestamp.strftime("%m.%d")

```

```python

from typing import List, Dict

from fiam.store.formats import EventRecord, Turn

from .dynamics import extract_dynamics, relative_time

def extract_raw_fragments(event: EventRecord) -> str:

    """

    高arousal事件：保留原文片段

    """

    if not event.turns:

        return ""

    

    # 取user第一句话（前80字符）

    user_first = event.turns[0].user_content[:80]

    if len(event.turns[0].user_content) > 80:

        user_first += "..."

    

    # 取assistant核心反应（第一个完整句子或前60字符）

    assistant_content = event.turns[0].assistant_content

    # 简单规则：找第一个句号/问号/感叹号

    for end_char in ['. ', '! ', '? ', '。', '！', '？']:

        if end_char in assistant_content:

            core = assistant_content.split(end_char)[0] + end_char.strip()

            break

    else:

        core = assistant_content[:60]

        if len(assistant_content) > 60:

            core += "..."

    

    return f'"{user_first}"\n我：{core}'

def extract_skeleton(event: EventRecord) -> str:

    """

    低arousal事件：规则生成骨架

    """

    if not event.turns:

        return "那次对话"

    

    # 提取主题关键词（简单规则：user第一句的前15字）

    user_first = event.turns[0].user_content

    topic = user_first[:15]

    if len(user_first) > 15:

        topic += "..."

    

    return f"那次关于「{topic}」的对话"

def prepare_materials(events: List[EventRecord]) -> List[Dict]:

    """

    为每个event准备材料

    """

    materials = []

    

    for event in events:

        # 根据arousal决定保留程度

        if event.arousal > 0.7:

            content = extract_raw_fragments(event)

        else:

            content = extract_skeleton(event)

        

        # 提取dynamics

        dynamics = extract_dynamics(event.turns)

        

        # 时间转化

        when = relative_time(event.timestamp)

        

        materials.append({

            "content": content,

            "dynamics": dynamics,

            "when": when,

            "arousal": event.arousal  # 仅用于内部判断，不传给LLM

        })

    

    return materials

```

### 3. 在 `synthesizer/narrative.py` 实现LLM调用

```python

import anthropic

from fiam.config import Config

def synthesize_narrative(materials: List[Dict], config: Config) -> str:

    """

    调用LLM生成第一人称叙事

    """

    if not materials:

        return ""

    

    # 构建prompt

    content_blocks = []

    for m in materials:

        block = f"{m['when']}：\n{m['content']}"

        if m['dynamics']:

            block += f"\n（{m['dynamics']}）"

        content_blocks.append(block)

    

    materials_text = "\n\n".join(content_blocks)

    

    system_prompt = """你在生成第一人称回忆片段。

不要：

- "用户"、"根据"、"记录"、"数据"

- "我应该"、"我需要"、"我会"

- meta视角（"作为AI"、"我的记忆系统"）

- 形容词堆叠（"深深的"、"强烈的"、"充满XX的"）

要：

- 第一人称："我"、"他"

- 像在回忆，不是在汇报

- 直接引用原话（如果材料里有引号）

- 可以模糊、不确定（"好像"、"记得"）

- 自然的时间感（"那次"、"后来"）"""

    user_prompt = f"""这些事忽然想起来了：

{materials_text}

以第一人称写一段回忆。150-250字。英文或中文（根据材料语言自然选择）。

像"想起那次..."的感觉，不要列表，不要分段。

直接写回忆内容，不要任何前缀或解释。"""

    # 调用Claude

    client = anthropic.Anthropic(api_key=config.api_key)

    

    response = client.messages.create(

        model="claude-3-5-haiku-20241022",  # 快速+便宜

        max_tokens=400,

        temperature=0.7,

        system=system_prompt,

        messages=[{

            "role": "user",

            "content": user_prompt

        }]

    )

    

    return response.content[0].text.strip()

```

from typing import List, Optional

from fiam.config import Config

from fiam.store.formats import EventRecord

from fiam.values.formats import ValueState

from .narrative import prepare_materials, synthesize_narrative

class StanceSynthesizer:

    """

    Synthesize narrative stance from retrieved events

    """

    

    def __init__(self, config: Config):

        self.config = config

    

    def generate(

        self,

        events: List[EventRecord],

        current_values: Optional[ValueState] = None

    ) -> str:

        """

        Generate first-person narrative from events

        

        Args:

            events: Retrieved events (from joint retriever)

            current_values: Current value state (optional, not used yet)

        

        Returns:

            Narrative text (150-300 words)

        """

        if not events:

            return ""

        

        # Prepare materials

        materials = prepare_materials(events)

        

        # Synthesize via LLM

        narrative = synthesize_narrative(materials, self.config)

        

        return narrative

from typing import List

from datetime import datetime

def extract_dynamics(turns: List[Turn]) -> str:

    """

    提取对话动态的自然语言描述

    检测：

    - 消息节奏（时间间隔）

    - 长度不对称（user vs assistant）

    - 沉默/突然爆发

    返回：自然语言片段（如 "消息很密集，他的话明显更长"）或空字符串

    """

    if len(turns) < 2:

        return ""

    # 节奏分析

    time_diffs = [

        (turns[i+1].timestamp - turns[i].timestamp).total_seconds()

        for i in range(len(turns)-1)

    ]

    avg_gap = sum(time_diffs) / len(time_diffs)

    pace = ""

    if avg_gap < 30:

        pace = "消息很密集"

    elif avg_gap < 120:

        pace = "节奏挺快"

    # 长度分析

    user_lengths = [len(t.user_content) for t in turns]

    assistant_lengths = [len(t.assistant_content) for t in turns]

    user_avg = sum(user_lengths) / len(user_lengths)

    assistant_avg = sum(assistant_lengths) / len(assistant_lengths)

    asymmetry = abs(user_avg - assistant_avg)

    length_note = ""

    if asymmetry > 100:

        if user_avg > assistant_avg:

            length_note = "他的话明显更长"

        else:

            length_note = "我说得更多"

    # 拼接

    parts = [p for p in [pace, length_note] if p]

    return "，".join(parts) if parts else ""

def relative_time(timestamp: datetime) -> str:

    """

    转化为相对时间表达

    """

    from datetime import datetime, timedelta

    now = datetime.now()

    delta = now - timestamp

    if delta < timedelta(hours=12):

        return "今天"

    elif delta < timedelta(days=1):

        return "昨天"

    elif delta < timedelta(days=7):

        return "前几天"

    else:

        return timestamp.strftime("%m.%d")

```

        return ""

    # 取user第一句话（前80字符）

    user_first = event.turns[0].user_content[:80]

        user_first += "..."

    # 取assistant核心反应（第一个完整句子或前60字符）

    assistant_content = event.turns[0].assistant_content

            core = assistant_content.split(end_char)[0] + end_char.strip()

            break

    else:

        core = assistant_content[:60]

        if len(assistant_content) > 60:

"""

记忆衰减和访问强化机制

基于Ebbinghaus遗忘曲线

"""

from datetime import datetime

import math

from typing import List, Tuple

from fiam.store.formats import EventRecord

def compute_retention(event: EventRecord, now: datetime) -> float:

    """

    计算记忆保留率

    公式：R(t) = e^(-t/S)

    - t: 距离事件发生的天数

    - S: 记忆强度（strength）

    - R: 保留率 [0, 1]

    Args:

        event: 事件记录

        now: 当前时间

    Returns:

        保留率（0~1，越接近1越"新鲜"）

    """

    days_since = (now - event.time).total_seconds() / 86400

    # strength决定衰减速度

    # strength=1.0: 14天后保留率约0.5

    # strength=2.0: 28天后保留率约0.5

    # strength=3.0: 42天后保留率约0.5

    decay_rate = days_since / event.strength

    retention = math.exp(-decay_rate / 14.0)

    return retention

def boost_strength(event: EventRecord, now: datetime) -> float:

    """

    计算访问后的新strength

    规则：

    - 基础增益：与arousal相关（高arousal事件增益大）

    - 递减因子：访问次数越多，增益越小

    - 新鲜度奖励：新事件（<7天）额外增益

    - 上限：3.0

    Args:

        event: 事件记录

        now: 当前时间

    Returns:

        新的strength值

    """

    # 基础增益（arousal越高增益越大）

    base_boost = 0.1 + (event.arousal * 0.2)  # 0.1~0.3

    # 递减因子（防止无限增长）

    diminishing = 1.0 / (1 + event.access_count * 0.1)

    # 新鲜度奖励（7天内的事件）

    days_since = (now - event.time).total_seconds() / 86400

    if days_since < 7:

        freshness_bonus = (7 - days_since) / 7 * 0.2  # 最多+0.2

        base_boost += freshness_bonus

    # 计算新strength（上限3.0）

    new_strength = min(event.strength + base_boost * diminishing, 3.0)

    return new_strength

def rerank_by_decay(

    events: List[EventRecord],

    base_scores: List[float],

    now: datetime,

    decay_weight: float = 0.3

) -> List[Tuple[EventRecord, float]]:

    """

    结合时间衰减重新排序

    最终得分 = base_score * (1 - decay_weight) + retention * decay_weight

    Args:

        events: 候选事件

        base_scores: 原始相似度得分

        now: 当前时间

        decay_weight: 衰减权重（0~1，0表示完全忽略时间）

    Returns:

        [(event, final_score), ...] 按final_score降序

    """

    scored = []

    for event, base_score in zip(events, base_scores):

        retention = compute_retention(event, now)

        final_score = base_score * (1 - decay_weight) + retention * decay_weight

        scored.append((event, final_score))

    scored.sort(key=lambda x: x[1], reverse=True)

    return scored

def apply_diversity_penalty(

    events: List[Tuple[EventRecord, float]],

    now: datetime,

    recent_days: int = 3,

    penalty: float = 0.7

) -> List[Tuple[EventRecord, float]]:

    """

    对最近访问过的事件降权（鼓励diversity）

    Args:

        events: [(event, score), ...]

        now: 当前时间

        recent_days: 多少天内算"最近"

        penalty: 惩罚系数（0~1，越小惩罚越重）

    Returns:

        调整后的 [(event, score), ...]

    """

    adjusted = []

    for event, score in events:

        if event.last_accessed:

            days_since_access = (now - event.last_accessed).total_seconds() / 86400

            if days_since_access < recent_days:

                score *= penalty

        adjusted.append((event, score))

    adjusted.sort(key=lambda x: x[1], reverse=True)

    return adjusted

def update_access_record(event: EventRecord, now: datetime) -> EventRecord:

    """

    更新事件的访问记录

    Args:

        event: 事件记录

        now: 当前时间

    Returns:

        更新后的事件（修改了strength, last_accessed, access_count）

    """

    event.strength = boost_strength(event, now)

    event.last_accessed = now

    event.access_count += 1

    return event

"""

多样性过滤：避免召回高度相似的事件

"""

import numpy as np

from typing import List

from fiam.store.formats import EventRecord

def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float:

    """计算余弦相似度"""

    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

def diversify_results(

    candidates: List[EventRecord],

    top_k: int = 5,

    similarity_threshold: float = 0.88

) -> List[EventRecord]:

    """

    简化的多样性选择：贪心去重

    逻辑：

    1. 第一个直接选（最相关的）

    2. 后续每个候选，检查是否和已选的"太相似"

    3. 如果相似度 > threshold，跳过；否则加入

    4. 直到选够top_k个

    Args:

        candidates: 候选事件（已按相关性降序排列）

        top_k: 要选的数量

        similarity_threshold: 相似度阈值（超过此值视为重复）

    Returns:

        去重后的事件列表

    """

    if len(candidates) <= top_k:

        return candidates

    selected = []

    for candidate in candidates:

        if len(selected) >= top_k:

            break

        # 第一个直接加

        if not selected:

            selected.append(candidate)

            continue

        # 检查是否和已选的任何一个太相似

        is_redundant = False

        for selected_event in selected:

            sim = cosine_similarity(candidate.embedding, selected_event.embedding)

            if sim > similarity_threshold:

                is_redundant = True

                break

        if not is_redundant:

            selected.append(candidate)

    return selected

            candidates.append((event, score))

        # 2. 应用时间衰减

        events_only = [e for e, _ in top_candidates]

        scores_only = [s for _, s in top_candidates]

        reranked = rerank_by_decay(

            events=events_only,

            base_scores=scores_only,

            now=now,

            decay_weight=self.config.decay_weight

        )

        # 3. 应用diversity penalty

        reranked = apply_diversity_penalty(

            events=reranked,

            now=now,

            recent_days=3,

            penalty=0.7

        )

        # 4. 多样性过滤

        events_after_decay = [e for e, _ in reranked]

        final_events = diversify_results(

            candidates=events_after_decay,

            self.vault.update_event_metadata(event)

# fiam

Bio-inspired affective memory for AI — 跨会话的情感记忆管道。

---

## 它是什么

fiam 是一个围绕 Claude Code 运行的后台 daemon。它监测对话日志，提取有情感显著性的事件，存入记忆，并在会话中实时注入相关回忆。

AI 不参与记忆管理。记忆的编码、存储、检索、遗忘全部由 fiam 在 AI 之外完成——像海马体一样，工作在意识之下。

---

## 理念

**不扮演，不伪装，不为了拟人而拟人。**

fiam 不伪造"人类式记忆"。AI 知道 recall 来自 hook 注入，正如人类知道自己会遗忘但无法阻止——这种透明是诚实的，不需要隐藏。

回忆的注入像 BGM：背景里浮现的东西，AI 可以回应，可以忽略，不是指令。形式上是 `additionalContext`，不是 user message，不会被混入对话流。

**拒绝 LLM 级联。** LLM 管理 LLM 的记忆会导致风格趋同、细节磨损、幻觉累积。fiam 用小模型分类 + 规则过滤做存储决策，LLM 只在叙事合成环节可选介入。

**事件而非画像。** 存储的不是"用户是怎样的人"，而是"那次发生了什么，当时什么感觉"。

**人格自写。** 没有结构化的价值观文件。AI 自己维护 `self/personality.md`，格式不限。

### 两条路径

**实时路径（会话中）**：daemon 检测到 JSONL 有新内容 → 提取最近用户文本 → 嵌入 → 与上次 recall 查询比较余弦相似度 → 低于 0.65 = 话题漂移 → 重新检索 → 重写 `recall.md`。UserPromptSubmit hook 在每次用户发送消息时读取 `recall.md` 注入为 `additionalContext`。

**批处理路径（会话后）**：空闲超时后 → 增量解析 JSONL（字节偏移） → 情感分类 → 显著性门控 → 存入事件 → 刷新 recall → 等待下一次活动。

### 显著性门控

事件通过任意一个信号即可存储（any-of）：

| 信号 | 阈值 | 含义 |

|---|---|---|

| emotional | arousal > 0.6 | 情绪显著 |

| novelty | > 0.7 | 与已有记忆语义距离大 |

| elaboration | > 1.5 | 用户文本长度 / 会话中位数（log2） |

Valence（情感极性）被存储但不参与门控。

### 记忆检索

```

base = 0.5 × cosine(query, event) + 0.3 × retention(event)

final = base + 0.2 × temporal_boost(event, top_candidates)

```

遗忘曲线：$R(t) = e^{-t\,/\,(S \cdot 14)}$，$S$ = 强度 ∈ [1.0, 3.0]，被召回时强化。

多样性过滤：贪心去重，跳过与已选事件余弦相似度 > 0.88 的候选，输出 top-5。

### recall.md 格式

```markdown

<!-- recall | 2026-04-06T09:54:20Z -->

- (3天前) 啊啊啊啊数据全丢了三周全白费了！！！ → 别急，先看看有没有备份

- (2小时前) 代码跑通了 → 恭喜！

- (14天前) I spent the entire weekend rewriting the memory system from scratch...

```

干净的记忆碎片，没有角色标记，没有对话格式。短交互用 `→` 连接用户与 AI 的核心语句，长文本只保留用户侧。

## Hook 机制

fiam 通过 Claude Code 的 `UserPromptSubmit` hook 实现会话中注入。hook 在每次用户发送消息时触发，读取 `recall.md`，作为 `additionalContext` 返回——进入 system/context 层，不是 user message。

```json

// .claude/settings.local.json

{

  "hooks": {

    "UserPromptSubmit": [

      {

        "type": "command",

        "command": "powershell -File path/to/inject.ps1"

      }

    ]

  }

}

```

AI 知道这些内容来自 hook。这不是 bug，是特性——不需要伪装记忆来源。

## Setup (Windows)

> fiam is tested on Windows. The hook script is PowerShell.

> On Linux/Mac, use `test_hook/inject.sh` as the hook instead of inject.ps1.

**Requirements**

- [uv](https://docs.astral.sh/uv/) — Python dependency manager

- [Claude Code](https://claude.ai/code) installed and on PATH

- An empty directory to serve as the AI's home (e.g. `F:\fiet-home`)

**1. Install dependencies**

```bash

uv sync

```

First run downloads ~1.2 GB of models to `~/.cache/huggingface/`. Subsequent starts use the cache.

> Do not use conda's torch on Windows. `uv run` uses `.venv` automatically.

**2. Run the setup wizard**

```bash

uv run python scripts/fiam.py init

```

Answers three questions (home path, AI name, your name), then creates:

```

fiam.toml                           config

{home}/CLAUDE.md                    read by Claude Code at startup

{home}/.claude/hooks/inject.ps1     the memory hook

{home}/.claude/settings.local.json  enables the hook in Claude Code

{home}/.gitignore, self/, {name}/   home structure

```

**3. Start Claude Code from the home directory**

```powershell

cd F:\fiet-home

claude

```

Claude Code must be started from the home directory so `CLAUDE_PROJECT_DIR` points there and the hook finds `recall.md`.

**4. Start the daemon (separate terminal)**

```bash

uv run python scripts/fiam.py start

```

fiam polls Claude Code's JSONL logs. On new activity it checks for topic drift and updates `recall.md` live. After 30 min of silence it runs the full post-session pipeline (emotion classification → event storage → memory refresh).
