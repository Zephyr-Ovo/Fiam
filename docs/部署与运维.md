### 为什么这么部署？

- **ISP 用家庭宽带** — 没有数据中心 IP 特征，CC 不会标记为自动化流量
- **DO 只做推理** — bge-m3 需要 ~1.5GB RAM，ISP 的 2GB 装不下。把嵌入卸到 DO
- **Relay 纯跳板** — 本地没法直连 ISP，LA 的 VPS 做 SSH ProxyJump
- **本地不跑 CC** — 只做开发。JSONL 通过 git 同步到 ISP

## 安全加固 (Day 3)

`01618fd` — feat(security): viewer token auth + bind 127.0.0.1; ISP SSH hardening notes

### SSH 加固

ISP 的 SSH 配置通过 drop-in 文件，不动主 sshd_config：

`/etc/ssh/sshd_config.d/99-fiam-hardening.conf`:
```
Port 52237
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
```

- **端口 52237** — 非标准端口，减少扫描噪声
- **禁止 root 登录** — 只有 `fiet` 用户
- **纯密钥认证** — 关闭密码登录
- **fail2ban** — maxretry=5, bantime=3600

### Dashboard 安全

- **绑定 127.0.0.1** — 不对外暴露
- **通过 SSH 隧道访问** — 本地 `ssh -L 8766:127.0.0.1:8766 isp`
- **Viewer token** — `FIAM_VIEW_TOKEN` 环境变量，`?token=xxx` 参数认证
- **Ingest token** — `FIAM_INGEST_TOKEN`，写操作需要更高权限

## DO 嵌入服务器

### 内存预算 (4GB RAM + 4GB swap)

| 模型 | 磁盘 | 内存 |
|------|------|------|
| bge-m3 | 4.3GB | ~1.5GB |
| go_emotions (lazy) | 479MB | ~500MB |
| Chinese-Emotion (lazy) | 1.1GB | ~800MB |

**关键经验：**

> bge-m3 在 CPU 上对长文本极慢（全 batch 文本 pad 到最大长度）。服务端超过 512 字符直接截断——100x 加速。

> Emotion 模型不 unload → bge-m3 被推到 swap → embed 调用从 3-5 秒变成 5+ 分钟 → TIMEOUT。
> 解法：`/unload_emotion` endpoint，批量 embed 前先卸载 emotion 模型。

## 成本控制

CC 是订阅制（不按 API 计费），但 Zephyr 仍然设了成本意识系统：

- 每次 `claude -p` 调用后记录 `total_cost_usd` 到 ledger
- `check_budget()` 在每次唤醒前检查——超出日预算自动拒绝唤醒
- 显示在 daemon console 上：`[yellow]budget: exceeded daily limit[/]`

> 📁 `scripts/fiam_lib/cost.py` · `2a7ea2f` — feat: quota awareness

## Scheduler

定时任务系统支持 AI 自己安排唤醒：

- AI 在回复中写 `[WAKE:2026-04-20T08:00:00 reason]` → scheduler 解析存储
- daemon 每个 poll 周期检查 `load_due()` → 到时间就执行唤醒
- 灾难恢复：grace window（过期 15 分钟内仍可触发）、重试、原子写入
- 去重：同一时间同一原因不重复创建

WAKE 类型：
| 类型 | 含义 |
|------|------|
| `private` | 静默后台任务，不通知用户 |
| `notify` | 执行后推送结果给用户 |
| `seek` | 尝试联系用户（TG → SSH/AW） |
| `check` | 静默环境观察 |

> 📁 `scripts/fiam_lib/scheduler.py` · `6d40967` — integrate scheduler into daemon
